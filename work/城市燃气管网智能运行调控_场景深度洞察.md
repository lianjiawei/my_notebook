# 城市燃气管网智能运行调控 — 场景深度洞察

本文聚焦城市燃气管网的**智能运行调控**场景，深入拆解其业务本质、技术链路和能力需求，为华为解决方案架构师提供可落地的洞察框架。不泛谈"智慧燃气"全景，而是围绕**管网工况感知→负荷预测→调度决策→应急保障**这条核心业务链展开。

---

## 一、业务本质：管网运行调控到底在做什么？

### 1.1 城市燃气管网的物理结构

城市燃气管网是一个**分级调压、树状+环状混合**的压力管道系统：

```
上游气源（门站接收）
  │
  ▼
高压管网（1.6-4.0 MPa）── 长输管道接入、LNG气化站接入
  │
  ▼  [高-中调压站]
次高压/中压A管网（0.4-1.6 MPa）── 城市主干管
  │
  ▼  [区域调压站/调压柜]
中压B管网（0.01-0.4 MPa）── 市政配气管
  │
  ▼  [楼栋调压箱/用户调压器]
低压管网（<0.01 MPa / <2kPa）── 入户管道
```

**核心物理约束**：
- 管网是"有压流"系统，压差驱动气体流动
- 调压器是被动设备，靠弹簧/膜片自动调节，非远程主动控制
- 管道本身有"管存"——可通过升压在管道中临时储存气量（管存调峰）
- 管网拓扑复杂，环网与枝状网并存，水力计算是非线性问题

### 1.2 调度中心的核心职责

调度中心是管网运行的"大脑"，24小时值守，核心工作链路：

```
数据采集 → 态势感知 → 负荷预测 → 调度决策 → 指令执行 → 效果验证
    ↑                                                    │
    └────────────────── 闭环反馈 ──────────────────────────┘
```

**调度员每天具体在做什么：**
1. **监盘**：通过SCADA看各门站/调压站的压力、流量、温度，判断管网是否正常
2. **记录与上报**：填写调度日报（日供气量、峰值流量、管网压力范围等）
3. **预判**：根据天气预报、历史数据、节假日等，预判明天/未来几天用气需求
4. **调配**：协调多个气源的供气量（上游管道气、LNG补充、CNG应急），指挥调压站调整设定压力
5. **应急处置**：管网泄漏/爆管/设备故障时，决策关阀方案、启动应急预案、协调抢修

## 二、管道工况数据：感知层的深度分析

### 2.1 工况数据的分类与采集

| 数据类型 | 具体参数 | 采集位置 | 采集方式 | 频率 |
|----------|---------|----------|---------|------|
| **压力数据** | 绝对压力、表压、压差 | 门站、调压站进出口、管网关键节点、末端 | 压力变送器→RTU→SCADA | 秒级~分钟级 |
| **流量数据** | 瞬时流量、累积流量 | 门站、分输站、大用户计量点 | 涡轮/超声波流量计→RTU | 分钟级 |
| **温度数据** | 气体温度、环境温度 | 门站、调压站 | 温度传感器 | 分钟级 |
| **组分数据** | 热值、密度、含水量 | 门站（气源入口） | 气相色谱仪/在线分析仪 | 小时级 |
| **设备状态** | 调压器阀位、过滤器压差、阴保电位 | 各场站、调压设施 | PLC/RTU | 实时 |
| **环境数据** | 可燃气体浓度 | 调压站/阀室/管廊 | 可燃气体探测器 | 实时 |

### 2.2 当前工况数据的核心痛点

**痛点1：采集覆盖率严重不足**
- 门站和高压管网监测较完善（100%覆盖）
- 中压管网的区域调压站/调压柜覆盖率通常只有30-60%
- 低压管网和末端几乎没有在线监测（"盲区"）
- 大量中小调压箱无通信条件（无电源、无网络信号）

**痛点2：数据质量差**
- 传感器长期不校准，数据漂移
- RTU通信不稳定，数据丢包、延迟
- 不同时期安装的设备协议不统一（Modbus/OPC/私有协议混杂）
- 缺乏数据清洗和异常值处理机制

**痛点3：数据利用率低**
- SCADA只做"看板"，不做分析
- 海量历史数据沉睡在数据库中，未被挖掘
- 工况数据与GIS空间数据、客户用气数据未关联
- 调度员依赖经验而非数据做决策

### 2.3 工况数据问题对调度决策的影响

| 数据问题 | 对调度决策的具体影响 | 华为的对应方案 |
|----------|-------------------|--------------|
| 中低压管网监测盲区 | 调度员无法掌握末端压力，不知道用户端是否供气正常；仿真模型缺少校准数据 | **双重补盲策略**：①物理补盲——NB-IoT低功耗传感回传关键监测点；②计算补盲——管网仿真引擎"软表"技术，在管网任意位置通过计算获得虚拟监测数据（湃孚管能大脑已实现此能力） |
| 数据协议不统一 | 多厂商SCADA/RTU数据无法在调度中心统一呈现，形成信息盲区 | ROMA Connect协议适配与数据归一化 |
| 数据质量差 | 脏数据输入导致负荷预测偏差大、仿真模型不准确、异常告警误报 | DataArts数据治理（清洗、补全、异常值处理） |
| 数据未关联 | SCADA工况数据与GIS拓扑、用户用气数据各自独立，无法支撑综合调度分析 | 数据湖+数据中台打通多源数据 |

## 三、用气负荷预测：调度决策的核心输入

### 3.1 负荷预测的业务场景

| 预测类型 | 时间尺度 | 用途 | 精度要求 |
|----------|---------|------|---------|
| **长期预测** | 年/季度 | 气源采购合同签订、管网扩容规划 | ±10-15% |
| **中期预测** | 月/周 | 月度供气计划、LNG采购计划、检修安排 | ±5-10% |
| **短期预测** | 日前/日内 | 次日供气量安排、调压策略制定 | ±3-5% |
| **超短期预测** | 小时级 | 实时调度微调、峰值应对 | ±5-8% |

### 3.2 影响负荷的关键因子

```
用气负荷 = f(气象因子, 时间因子, 社会因子, 特殊事件)

气象因子（权重最大，尤其冬季采暖地区）：
  ├── 日均温度（最关键，与用气量强负相关）
  ├── 风速/风力（影响体感温度）
  ├── 日照时长
  └── 降雨/降雪

时间因子：
  ├── 时刻（日内曲线：早晚高峰、午间低谷、夜间低谷）
  ├── 星期几（工作日 vs 周末模式不同）
  ├── 月份/季节（采暖季 vs 非采暖季差异巨大，可达3-5倍）
  └── 是否假期

社会因子：
  ├── 区域人口/用户数增长
  ├── 工业用户开工率
  ├── 新接驳用户数
  └── 区域GDP/经济活跃度

特殊事件：
  ├── 寒潮/极端天气（用气量可能突增30-50%）
  ├── 重大赛事/活动
  ├── 上游气源限量/中断
  └── 管网检修/改造导致的区域停气
```

### 3.3 预测技术现状与趋势

| 方法 | 特点 | 局限 |
|------|------|------|
| **经验法** | 调度员凭经验估算，当前主流 | 完全依赖个人，无法标准化，退休后知识流失 |
| **统计模型** | 回归分析、ARIMA时序模型 | 难以捕捉非线性关系，极端工况预测差 |
| **机器学习** | BP神经网络、随机森林、XGBoost | 需要较多特征工程，对数据质量要求高 |
| **深度学习** | LSTM、Transformer、集成学习 | 当前研究热点，对计算资源和数据量要求高，但精度最优 |
| **混合模型** | EMD分解+LSTM等组合方法 | 学术界验证效果好，工程落地还需简化 |

#### 3.3.1 负荷预测工程化落地的关键挑战（深度拆解）

学术论文中的高精度模型往往无法直接落地，工程化需要解决以下问题：

**数据准备阶段：**
```
原始数据 → 清洗 → 特征工程 → 模型输入

关键难点：
1. 历史数据质量差：SCADA数据有大量缺失/异常值，需要智能补全
2. 气象数据获取：需要对接气象API，获取预报级（而非观测级）温度数据
3. 特殊日历标注：春节/国庆等假期模式需要人工标注，各地差异大
4. 用户结构变化：新接驳大用户、工厂停产等引起的结构性变化难以自动识别
5. 采暖季/非采暖季需要分别建模（用气模式本质不同）
```

**模型选择与训练：**
```
推荐的工程化方案（平衡精度与可维护性）：

方案A（推荐入门）：XGBoost + 手动特征工程
  ├── 优势：可解释性强、训练快、不需要GPU
  ├── 精度：日前预测误差约3-5%
  └── 适合：数据质量一般、IT能力弱的中小城燃

方案B（推荐进阶）：LSTM + 注意力机制
  ├── 优势：自动学习时序特征、精度高
  ├── 精度：日前预测误差约2-4%
  └── 适合：数据基础好、有AI团队的头部城燃

方案C（前沿探索）：时序大模型（预训练+微调）
  ├── 优势：少样本学习、跨区域迁移能力强
  ├── 精度：有待大规模验证
  └── 适合：华为盘古模型的差异化切入点
```

**模型部署与运维：**
```
训练环境 → 模型导出 → 推理部署 → 持续监控 → 定期重训

关键要求：
- 每日凌晨自动运行次日预测，结果推送至调度系统
- 预测结果需附带置信区间（不是给一个点值，而是给出范围）
- 模型漂移监测：当预测误差持续偏大时自动告警
- 极端天气时自动切换至保守预测策略
- 调度员可手动修正预测值（人机协同，而非完全替代）
```

### 3.4 华为AI能力的对接点

- **ModelArts**：提供模型训练、调优、部署全流程平台，支持上述方案A/B/C的全生命周期管理
- **盘古时序大模型**：预训练时序模型可迁移至燃气负荷预测，降低建模门槛。核心优势是**少样本学习**——新城燃客户不需要积累多年数据即可获得可用模型
- **算力支撑**：Atlas计算平台提供训练算力，边缘Atlas支持推理部署
- **关键差异化**：不是简单提供AI平台，而是与伙伴共建**燃气负荷预测行业模型**，形成"开箱即用"的行业解决方案

## 四、调度决策：从"人工经验"到"人机协同"

### 4.1 调度决策的核心问题域

调度员每天要回答的核心问题：

```
1. 供需平衡：明天（这周）需要多少气？各气源各供多少？
2. 压力调控：各级管网的目标压力设多少？哪些调压站需要调整？
3. 管存利用：夜间低谷时是否升压储气？高峰前何时开始释放？
4. 峰谷调节：尖峰时段LNG气化站是否启用？启用多少？
5. 异常应对：某门站压力下降/流量异常，如何调整其他气源补偿？
6. 检修协调：某段管道停气检修，如何调整周边管网供气路由？
```

### 4.2 管网仿真——调度决策的"沙盘推演"（深度拆解）

管网仿真是智能调度的核心技术能力，分为：

| 仿真类型 | 说明 | 应用场景 |
|----------|------|---------|
| **离线仿真** | 基于设计参数的稳态/动态计算 | 管网规划设计、扩容方案评估 |
| **在线仿真（关键）** | SCADA实时数据驱动的动态仿真 | 实时工况推演、调度方案验证 |
| **培训仿真** | 模拟各种事故场景 | 调度员培训、应急演练 |

**在线仿真的技术链路：**
```
SCADA实时数据 ──┐
GIS管网拓扑 ────┤──→ 水力仿真引擎 ──→ 全网压力/流量分布 ──→ 调度建议
用户用气数据 ──┘     （稳态/瞬态求解）    （包括无监测点位置）
```

**在线仿真解决的关键问题：**
- **看不到的地方也能"算"出来**：管网只有少数节点有传感器，仿真可推算全网任意位置的压力流量
- **"如果...会怎样"**：调度员可模拟"如果关掉某个气源/打开某个阀门"的影响
- **方案比选**：自动生成多个调度方案并比较优劣

**国产仿真引擎的实际应用场景（以湃孚管能大脑为例）：**

湃孚的产品架构将仿真能力组织为四大场景域，覆盖了调度全链路：

| 场景域 | 具体功能 | 对调度的支撑价值 |
|--------|---------|-----------------|
| **在线仿真计算** | 输配工况稳态计算、在线动态仿真、多气源溯源、输差计算、软表计量监测 | 实时掌握全网工况（含无传感器点位），通过"软表"补充监测盲区 |
| **沙箱操作与分析** | 在线拓扑分析、辅助规划设计、输配计划推演、应急预案推演、虚拟调度 | 在隔离环境中安全模拟任何操作的影响，为调度决策提供"沙盘推演" |
| **安全监视与故障诊断** | 超压/欠压/超速分析、异常工况诊断、泄漏检测、老化度识别、堵塞检测、传感器故障诊断 | 从被动告警升级为主动诊断，AI+仿真结合实现智能故障定位 |
| **调度优化与决策支持** | 调度方案寻优、多气源流量/压力优化、冬季保供方案、成本结构优化、管存量分析、放散分析 | 直接输出可执行的调度优化建议，支撑人机协同决策 |

**关键技术亮点——"软表"概念：**
在管网任意位置插入虚拟计量点（软表），通过仿真计算获得该位置的压力、流量、流速、温度等数据。这一能力：
- 本质上解决了"中低压管网监测盲区"问题——不是靠增加物理传感器，而是靠计算补盲
- 可临时替代故障传感器，保障数据连续性
- 与华为NB-IoT物理传感方案形成**"物理补盲+计算补盲"的互补策略**

#### 4.2.1 仿真引擎的三大基石：算法、算力、数据

**算法层面（核心求解器）：**
- 数学基础：连续性方程 + 动量方程 + 能量方程 + 气体状态方程，构成非线性偏微分方程组
- **稳态求解**：给定边界条件（气源压力/流量、用户需求），求解全网平衡状态。适合规划设计、慢变化工况分析
- **瞬态求解**：考虑时间维度，模拟管网从一个状态过渡到另一个状态的动态过程（例如：关阀后压力波如何传播、管存如何变化）。适合应急模拟、调峰分析
- **状态估计**：利用有限的SCADA实测数据，通过最小化测量误差的加权残差，推算全网状态。这是在线仿真的关键技术——类似电力系统的状态估计，但天然气的可压缩性使问题更复杂
- 关键难点：节点数量大（城市管网可达50万-270万管段）、非线性强、实时性要求高

**数据层面（模型的"燃料"）：**
```
管网仿真需要三类数据的协同：

1. 拓扑数据（来自GIS）—— 静态，但需持续更新
   ├── 管段：管径、长度、材质、粗糙度、埋深
   ├── 节点：三通、弯头、阀门位置
   ├── 设备：调压器特性曲线、压缩机性能曲线
   └── 用户接入点：位置、接入管径

2. 实时运行数据（来自SCADA）—— 动态，秒级/分钟级
   ├── 边界条件：门站进站压力/流量
   ├── 中间监测点：调压站进出口压力/流量
   └── 设备状态：阀门开关状态、调压器设定值

3. 需求数据（来自营收/客服系统）—— 半静态
   ├── 各区域用户数量和类型
   ├── 大用户合同用气量
   └── 历史用气模式（日负荷曲线）
```

**算力层面：**
- 稳态计算：普通服务器即可，秒级完成
- 瞬态计算：计算量随管段数量和时间步长急剧增加，大规模管网需要高性能计算
- 在线仿真：需要准实时（分钟级刷新），且需高可靠性（不能中断）
- **国产仿真引擎的算力实践**：湃孚科技管能大脑已支持云计算架构、高性能并行计算、显卡异构计算，并与国家超算郑州中心联建实验室，千公里级管网可实现分钟级求解
- **华为算力切入点**：鲲鹏服务器为仿真引擎提供算力底座，Atlas GPU异构计算加速大规模管网求解，边缘Atlas支持场站级局部仿真。湃孚管能大脑的云计算/并行计算/GPU异构计算架构，天然适合部署在华为算力平台上

#### 4.2.2 模型校准——仿真精度的生命线

在线仿真模型的精度取决于模型与实际管网的匹配程度，需要持续校准：

```
校准流程：
1. 将SCADA实测值与仿真计算值对比
2. 识别偏差较大的区域
3. 调整模型参数（管道粗糙度、用户需求分配、阀门系数等）
4. 反复迭代直到偏差在可接受范围内（通常压力偏差<5%）

自动校准的挑战：
- 参数空间巨大（每根管道都有粗糙度参数、每个节点都有需求参数）
- 多解问题：不同参数组合可能产生相似结果
- GIS数据不准确会导致校准无法收敛

→ AI在此处的价值：机器学习辅助参数辨识，大幅缩短校准周期
```

### 5.3 管存调峰——容易被忽略的关键能力（深度拆解）

**什么是管存调峰：**
管道本身就是一个"储气罐"。在夜间用气低谷时，维持较高的管网运行压力，将多余的气"存"在管道中；在白天用气高峰时，允许管网压力适度下降，"释放"管存气量来补充供气。

#### 5.3.1 管存量的计算原理

```
管存量（标准方况） = V × P / (Z × T × P₀ × T₀⁻¹)

其中：
  V = 管道内部容积（由管径和长度决定，是固定值）
  P = 管道内绝对压力（运行中可变）
  Z = 气体压缩因子（与压力、温度、气体组分相关）
  T = 气体温度
  P₀, T₀ = 标准状态参数

简化理解：
  管存变化量 ≈ V × (P高 - P低) / (Z × R × T)

  即：压力变化1个单位，管存变化量与管道总容积成正比
```

**一个典型案例的量化分析：**
```
假设某中型城市管网参数：
  中压管网总长度：3000 km
  平均管径：DN200（内径约200mm）
  管道总容积：V ≈ π × 0.1² × 3,000,000 = 94,248 m³ ≈ 9.4万m³

  运行压力范围：0.15-0.35 MPa（中压B管网）
  可调压力区间：0.20-0.30 MPa（安全运行区间）

  管存变化量 ≈ 9.4万 × (0.30 - 0.20) × 10⁶ / (0.85 × 8314 × 288)
            ≈ 约 4.6万 Nm³

  如果日高峰缺口为 10万Nm³/h，管存可支撑约 0.5小时
  虽然时间不长，但省下了同等量LNG气化的成本

  经济账：
  LNG调峰价格 ≈ 4-5元/Nm³（含气化成本）
  管道气价格 ≈ 2-3元/Nm³
  管存调峰仅有管道折旧成本（边际成本几乎为零）

  每天利用管存调峰4.6万Nm³ → 节约 (4.5-2.5) × 46000 = 9.2万元/天
  一个采暖季(120天) → 节约约 1100万元
```

#### 5.3.2 管存调峰智能优化的数学模型

```
优化目标：
  min 总调峰成本 = f(LNG用量, 管存利用量, 各气源配比)

约束条件：
  1. 供需平衡：各时段总供气量 ≥ 预测负荷量
  2. 压力约束：P_min ≤ 各节点压力 ≤ P_max（安全运行区间）
  3. 末端压力：用户端压力 ≥ 最低保障值（通常≥0.01 MPa）
  4. 气源约束：各门站供气量 ≤ 合同上限
  5. LNG约束：气化能力 ≤ 设备最大气化量
  6. 管存连续性：管存量在相邻时段需连续（不能"跳变"）

决策变量：
  - 各门站各时段的供气量
  - 管网各区域的目标压力
  - LNG气化站各时段的启停和气化量

求解方法：
  - 传统方法：线性规划/非线性规划（需简化管网模型）
  - AI方法：深度强化学习（将调度建模为序贯决策问题）
  - 混合方法：仿真引擎（物理模型）+ 优化算法（数学模型）联合求解
```

**华为的发力点**：
- **盘古优化大模型**可应用于此类多约束优化问题
- 结合管网仿真引擎（伙伴提供）+ AI优化算法（华为AI平台），形成**"仿真+优化"联合方案**
- 核心价值可量化：每年为中型城燃企业节约调峰成本**数百万至上千万元**

## 六、调度场景中容易被忽略的关键维度

### 6.1 调度与上下游系统的数据联动
- **与营销系统联动**：大用户用气量异常变化直接影响管网压力平衡；新大用户接入需调度侧评估管网承载能力
- **与气源合同系统联动**：调度决策需实时知道各气源的合同剩余量、价格梯度、违约成本
- **与检修系统联动**：停气检修的时间/范围直接约束调度方案；检修完成后的恢复供气也需调度协调
- **结论**：智能调度系统不是孤岛，数据集成能力（华为ROMA Connect）是关键使能

### 6.2 多气源调度的复杂性升级
- 从单一管道气源 → 多气源并存（管道气+LNG+CNG+分布式天然气）
- 不同气源的**气质差异**（热值、密度不同）导致计量修正问题——影响供需平衡计算精度
- 多气源的**经济性排序**：管道气（最便宜）→ 管存调峰（几乎免费）→ LNG（最贵），调度优化需考虑成本
- 部分城市开始接入**氢气掺混**，对调度模型和安全约束提出全新要求

### 6.3 调度人才断层——AI辅助调度的根本驱动力
- 优秀调度员需5-10年培养，核心知识靠"师徒传承"
- 行业老龄化严重，经验丰富的调度员即将退休
- **AI辅助调度的核心价值**：将老调度员的经验沉淀为可复用的知识和模型
- **培训仿真系统**可加速新人培养（虚拟调度场景模拟，替代多年实操积累）

### 6.4 调度系统的安全合规约束
- SCADA/调度系统有**等保三级**要求，需网络隔离、数据加密、审计留痕
- 大部分客户不接受公有云方案——调度数据必须留在企业内部
- 调度系统需要**高可靠性**（99.99%以上），宕机直接影响供气安全
- **华为云Stack私有云+鲲鹏信创**在此场景有天然合规优势

### 6.5 管网GIS数据——仿真精度的"数据地基"
- 很多老城区管网GIS数据不准确（历史管线位置、管径、材质信息缺失或错误）
- GIS数据不准 → 仿真模型不准 → 调度建议不可信 → 调度员不敢用
- 管网普查/补测是前置工作，**不是ICT层面能解决的，但必须在方案中正视**
- 建议方案中包含"数据治理+模型校准"专项阶段
- **湃孚的实践经验值得借鉴**：湃孚在落地实施中已形成一套应对基础数据不完善的方法——利用有限资料建立尽可能完善的模型，通过仿真试算对模型、气源、用户数据进行校验核对，协助企业完成资料梳理；对缺失信息化能力的节点进行流量/压力的模拟替代；利用AI算法对历史计量数据进行校核，精准查找误差较大的计量表具。**这种"边建边治"的务实方法比"先治理数据再上系统"的理想路径更具可行性**

### 6.6 与电力调度的类比与借鉴
- 电力调度（EMS/DMS）成熟度远高于燃气调度，技术理念可借鉴：
  - 电力状态估计 → 燃气管网状态估计
  - 电力潮流计算 → 管网水力计算
  - 电力无功优化 → 管网压力优化
  - 电力AGC自动发电控制 → 燃气自动调压控制（远期目标）
- 但燃气管网的**可压缩流**特性与电力（不可压缩的电流）本质不同，不可简单照搬
- **华为在电力调度领域的积累可复用部分平台和方法论**

### 7.2 华为+湃孚联合方案的集成架构

湃孚管能大脑提供三种部署策略，与华为平台的集成方式各异：

```
部署方式1：全量部署（湃孚整体交付）
  └── 湃孚提供完整系统（仿真引擎+业务服务+应用界面）
  └── 华为提供：云Stack基础设施 + 鲲鹏算力 + 网络
  └── 适合：新建项目，客户无现有调度系统

部署方式2：核心引擎+业务服务（推荐的华为联合方案）
  └── 湃孚提供：仿真核心引擎 + 管能云业务服务接口
  └── 华为提供：云Stack + 算力 + ROMA Connect数据集成 + DataArts数据治理 + ModelArts AI平台
  └── 第三方/客户提供：应用界面（可对接已有SCADA/调度系统）
  └── 适合：已有SCADA的改造升级项目（主流场景）

部署方式3：核心引擎（最灵活）
  └── 湃孚仅提供仿真计算求解器的HTTP服务接口
  └── 华为/第三方自行开发数据适配和业务应用
  └── 适合：深度定制项目，或华为希望主导应用层的项目
```

**关键集成接口**：湃孚核心引擎通过HTTP标准接口暴露以下计算服务——模型管理、稳态计算、动态计算、物性计算、综合计算等，华为ROMA Connect可直接对接这些接口实现数据编排和服务调度。
